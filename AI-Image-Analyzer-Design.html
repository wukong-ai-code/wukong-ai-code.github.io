1产品竞争力：竞品 · 差异化 · 留存 · 商业模式

技术可行不等于产品成立。本章回答"要不要做、凭什么赢、用户为何持续用、如何赚钱"——这是初稿缺失、经自评补齐的核心。

赛道最大的两个"死因"（必须正面应对） ① 准确性预期落差：AI 营养数字是估算（识别×份量×映射误差累乘），用户若期待"精确热量计"必然失望； ② 记录留存差：饮食记录类产品头号死因是"用户懒得每餐拍"。本方案把留存列为北极星，而非把识别精度当唯一目标。

1.1 竞品与差异化

类型代表短板我们的差异化
记录/热量类薄荷健康、Keep 饮食、Yazio多靠手动搜索录入，拍照识别弱、无个体化建议闭环拍照即得 + 结合个体状态的可执行建议
AI 拍照类Cal AI 等西式食物为主，中式菜/食材识别弱，营养库非本土中式菜 + 中国食物成分表本土化
专业/慢病医院营养科系统重、贵、非日常轻量日常 + 慢病模块，B/C 兼顾

差异化定位一句话："最懂中式家常菜、并把营养换成结合你身体状况的下一步行动的拍照助手"——不做"最准的热量计"，做"最有用的膳食教练"。

1.2 留存策略（北极星驱动）

降低记录门槛

  • 拍照 3 秒出结果，一键确认，绝不逼用户填表。
  • 常吃菜"一键复记"；支持一天补记。
  • 识别不准时用"选择/微调"代替"重填"。

制造回访理由

  • 每餐后即时、个性化的一句话建议（即时反馈）。
  • 每日/每周营养趋势与目标达成度。
  • 轻量目标与连续记录激励（streak）。

1.3 商业模式与北极星

C 端

基础免费 + 会员订阅（慢病方案、深度报告、无限历史）。

B 端

医院营养科/体检/企业食堂 API 与报表；健康管理机构白标。

北极星指标

次周留存与周人均记录餐数为核心；识别精度、建议满意度为支撑指标。

5知识沉淀架构：RAG 之外的更优解

核心洞察：RAG 不该被"替换"，而应被"分层"。 RAG / Skills / 长期记忆 / LLM Wiki 不是竞品，而是承载不同「知识类型」的不同工具。纯 RAG 方案的错误在于把"流程"和"个体经验"硬塞进了"查事实"的工具。

5.1 按知识类型选载体

知识类型本项目示例最佳载体为什么不用 RAG
权威事实/数值药典条目、食物成分表、DRIs 参考值RAG ✅这正是 RAG 主场：可引用、可追溯、可热更新、抗幻觉
程序性专家经验（怎么判断）病害诊断流程、膳食评估 SOP、鉴别决策树SkillsRAG 只给文本碎片，模型仍需自组织判断，不稳定；Skill 固化流程本身
个体/情境记忆用户过敏史/口味、地块历史病史、趋势长期记忆RAG 无状态、全局，无法沉淀渐进的个体经验
演进的经验共识疑难 case 结论、内部判定标准LLM WikiRAG 库缺治理（谁审/版本/冲突）；Wiki 给专家可编辑可审校入口

5.2 三条判断轴

① 是什么 vs 怎么做

陈述性→RAG/Wiki；程序性→Skills。专家经验的精华恰是程序性的，这是纯 RAG 最大短板。

② 全局 vs 个体

全局共享→RAG/Skills/Wiki；随交互积累的个体→长期记忆。

③ 谁维护/多久变

机器自动更新→RAG 向量库；需人工审校治理→Wiki（版本+审阅流）。

5.3 四层知识体系（Wiki 作治理入口）

🏛️ LLM Wiki — 真相源 / 治理层
专家（营养师/植保师）编辑与审校的入口，带版本与审阅流。向下同时供给 RAG（索引其内容）与 Skills（提炼为规程）。解决 RAG 库"谁维护、如何治理"的空白。
📚 RAG — 查事实
药典、食物成分表、DRIs、防治手册数值。向量检索 top-k，输出带可追溯引用，抗幻觉。
🧭 Skills — 走流程
诊断/评估的程序性规程封装为可复用技能，按需渐进加载；比检索文本更稳定地执行专家判断逻辑。
🧠 长期记忆 — 存个体
跨会话持久化用户/地块的情境经验：过敏忌口、口味、历史趋势、既往病史。让建议"记得你"，逐步个性化。
一句话总结RAG 存事实、Skills 存流程、记忆存个体、Wiki 做治理。四者协同，而非二选一。

6两个技术攻坚专题（决定成败）

初稿把这两点用"一句话"糊过去了，实际是 M1 最大的误差源与工程风险，单列攻坚。

6.1 菜品 → 食材 → 营养 的映射

难点成分表按食材存（鸡蛋/番茄/油分开），识别出的是成品菜（番茄炒蛋）。需完成：菜→食材拆分、份量在食材间分配、烹饪吸油/吸糖量估计（最难，缺权威数据）。
  • 建"标准菜谱-食材配比库"：常见中式菜的典型食材构成与比例，作为拆分先验（人工+营养师维护，进 Wiki 治理）。
  • 烹饪修正系数表：按烹饪方式（炒/炸/蒸/煮）附加油脂/糖分修正区间，而非点值。
  • MLLM 直接输出食材级份量作为补充，与配比库交叉校验。
  • 误差透明：营养值一律给区间，标注"估算"。

6.2 份量估计

难点单张随手拍中，餐具尺寸未知、俯拍角度、遮挡都会毁掉估计——全行业未解难题。
  • 标准餐具库：以常见碗/盘/勺尺寸为参照做体积→质量换算。
  • 对话式补充：低置信时追问"用的什么碗/大概几口"，用最小负担换精度（调研：补充食材质量可显著降误）。
  • 区间而非点值：份量、营养全程区间输出，管理用户预期。
  • P1 进阶：双视角/深度传感器。
价值主张诚实降级对外定位为"膳食趋势管理工具"而非"精确热量计"——避免准确性预期落差这一赛道死因。
