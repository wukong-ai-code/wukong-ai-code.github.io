1场景一 · 杭白菊种植管理智能分析

面向杭州及周边杭白菊种植科研与田间生产，以手机巡田、无人机航拍、田间定点相机采集的图像，实现生长态势分析、早期病虫害识别、产量预测三大任务，支撑精准种植与及时防治决策。

1.1 领域背景与任务边界

种植管理的核心痛点

  • 长势靠人工巡查：大田尺度长势不均难以量化，缺乏客观、可追溯的态势评估手段。
  • 病害发现滞后：叶枯/斑枯病、白锈病、霜霉病及蚜虫等一旦显性暴发已错过最佳防治窗口，早期弱信号识别价值最高。
  • 产量凭经验估计：杭白菊花期集中、采收窗口短，产量与采收时机预估直接影响用工与效益。

三类可落地任务

任务输出视觉可行性
T1 生长态势分析物候期识别 + 冠层覆盖度/现蕾密度 + 长势分级（旺/中/弱）高 冠层与物候特征明显
T2 早期病虫害识别病害类别 + 病斑定位/严重度 + 虫害预警中高 早期弱信号靠时序对比增强
T3 产量预测花蕾/花朵计数 → 单位面积产量与采收窗口中 计数 + 时序 + 气象融合
T4（可选）胁迫诊断缺素/水分胁迫致叶色黄化等中 多光谱可显著增强
诚实的技术边界 土壤养分、有效成分含量、病原菌种类无法从 RGB 图像确诊，须结合土壤检测、镜检/分子鉴定。图像模型定位于可见长势与病症的早期筛查与量化，输出"疑似 + 建议复检/防治"，不替代植保诊断。无人机多光谱（NDVI 等）可增强长势与胁迫判读，但增加成本，按需分级引入。

1.2 数据方案

多源 + 时序采集

  • 手机巡田：近景叶片/花蕾特写，用于病害识别与计数标注。
  • 无人机航拍：小区/大田尺度冠层正射，可选多光谱通道计算植被指数（NDVI/GNDVI）。
  • 田间定点相机：固定机位定时拍摄，构建同一地块的生育期时序序列——早期病害与长势变化的关键。
  • 元数据：地块、生育期日期、品种、农事与气象记录（用于产量建模）。

规模与标注

  • MVP 目标：病害每类 ≥ 800 张（含早期/中/重分期）+ 充足健康对照；计数场景 ≥ 1k 张带花蕾点/框标注。
  • 标注：病害类别 + 病斑分割/严重度、花蕾花朵实例框、长势分级；植保/农艺专家仲裁。
  • 时序标注：同一地块跨期序列，支撑早期弱信号与产量回归。
  • 冷启动：DINOv2 zero-shot + 少量种子样本主动学习，优先送标最不确定样本。

1.3 模型架构（检测/分类 + 时序 + RAG）

01多源采集/预处理航拍拼接配准、植被指数、去背景
02检测/分割YOLOv8 花蕾计数 + 病斑定位分割
03分类/分级DINOv2 病害细粒度 + 长势分级
04时序融合长势曲线 + 气象 → 产量/采收窗口回归
05防治 RAG对齐栽培规程/防治手册，出农艺建议
06决策 + 复核预警/建议 + 依据 → 农艺专家闭环

选型理由

  • 骨干：DINOv2（自监督）——田间小样本、细粒度病症下特征优于从头训练 CNN；冻结骨干 + 轻量微调即可起步，契合调研报告"迁移学习起步、小样本友好"的结论。
  • YOLOv8 实例分割——花蕾/花朵密集计数与病斑定位，支撑产量估计与严重度量化。
  • 时序建模是产量与早期病害的关键——将定点相机的长势指标序列 + 气象数据输入回归/时序模型，预测单产与采收窗口；早期病害靠"同地块跨期对比"放大弱信号，优于单帧判读。
  • 防治 RAG——把杭白菊栽培技术规程与病虫害防治手册向量化入库；识别出病害后检索对应条目，生成"疑似白锈病早期 → 及时通风控湿、参考 XX 药剂"式可追溯的农艺建议。
  • 可选 MLLM 复核层——疑难/低置信病症调用 Qwen-VL / GPT-4o 结合检索文本做二次判读与自然语言解释。

2场景二 · 日常膳食营养健康管理

面向中国家庭日常餐食，构建"拍照 → 识别菜品/原材料 → 份量估计 → 营养成分核算 → 结合个体状态的日常膳食建议"的对话式管线。核心遵循调研结论：MLLM + RAG 锚定权威营养库，并针对中式餐食的特殊难点做工程强化。

2.1 中式餐食的特殊难点

混合与共享

一盘多食材（回锅肉含肉/青椒/蒜苗）、圆桌共享餐无法按人分份，需"整盘估计 + 摄入占比追问"。

烹饪隐性热量

油炒/红烧带入大量看不见的油脂糖分，同一食材热量差异大，须引入"烹饪方式"维度修正。

份量与餐具

碗/盘/勺尺寸差异大，单视角深度歧义；以标准餐具库 + 参照物 + 可选多视角降低误差。

2.2 模型架构（对话式 MLLM + RAG）

01MLLM 视觉理解菜品/食材/烹饪方式/餐具识别
02份量估计餐具参照+（可选多视角/深度）→ 克重
03RAG 对齐中国食物成分表检索匹配
04营养计算热量+宏量/微量+GI 汇总
05对话澄清按需追问食材/份量提升精度
06个性化建议对齐膳食指南+慢病目标

关键设计

  • 识别层用 MLLM（Qwen-VL / GPT-4o）而非纯检测网络：直接输出结构化 JSON（菜名、食材列表、估计份量、烹饪方式、置信度），对开放域中式菜更鲁棒——契合调研中 GPT-4o 取得最低整体 MAPE 的结论。[17][19]
  • RAG 锚定《中国食物成分表》：不依赖模型记忆的营养数值，而是把识别出的食材名检索本地化营养库（对齐 FNDDS/USDA 思路，但换成中国本土数据），杜绝幻觉、保证可复现。[16]
  • 渐进式澄清（agentic）：默认仅凭图像给估算并标注置信区间；用户可选追加"食材/总重量"，误差随信息补充下降（调研：热量误差从 ~51% 图像-only 降到 ~29% 提供食材质量）。[17]
  • 份量估计：以图中标准餐具尺寸为参照做体积→质量换算；进阶支持双视角或手机深度传感器。

2.3 营养计算与个性化建议引擎

营养核算

  • 输出：能量（kcal）、三大宏量（碳水/蛋白/脂肪）、关键微量（钠/钙/铁/膳食纤维）、估算血糖生成指数。
  • 误差呈现：每项给点估计 + 区间，透明传达不确定性，避免"伪精确"。

个体状态驱动的建议引擎（RAG on 指南）

  • 个体状态档案：年龄/性别/身高体重(BMI)/活动量/健康目标/慢病标签/过敏与忌口 → 计算个性化能量与营养素目标（RNI/AI）。
  • 锚定《中国居民膳食指南 2022》八条准则与膳食宝塔，做当日摄入/趋势对照，给出"多吃·少吃·替换"的日常膳食建议。
  • 慢病模块：糖尿病（控糖/GI）、高血压（限钠）、减重（能量缺口）差异化建议。
  • 建议均附来源条目，保持可追溯，医学建议加免责声明。

3共性技术底座

两条产品线共享同一套"视觉基础模型 + RAG + 数据闭环"平台，仅知识库与分类头不同，最大化复用、降低边际成本。

统一技术栈

层选型
视觉骨干DINOv2 / CLIP（自监督基础模型）
检测分割YOLOv8 / Mask R-CNN
多模态推理Qwen-VL / GPT-4o（结构化输出）

数据闭环（护城河）

  • 专家反馈回流：质检员/营养师的每次修正即新标注，回流训练集。
  • 主动学习：优先送标模型最不确定 / RAG 无命中的样本。
  • 影子评测：新模型先旁路运行，与线上/专家对比达标再灰度上线。
  • 知识库热更新：药典/成分表/指南修订时只更新 RAG 库，无需重训模型。
复用要点同一套检索与推理编排，杭白菊换"药典库+品种分类头"，膳食换"成分表库+菜品识别"，工程主体不变。

4分阶段实施路线图

沿用调研报告的演进路径：迁移学习 MVP → MLLM+RAG 增强 → 专家标注反馈闭环。两条产品线并行推进，共享底座。

阶段一MVP 验证
  • 杭白菊：田间多源采集协议定稿；采集病害+长势种子数据；DINOv2 微调做 T2 病害分类 + T1 长势分级，验证早期召回可行性。
  • 膳食：接入 MLLM 出结构化识别；接《中国食物成分表》做营养核算原型；验证图像-only 误差水平。
  • 产出：两个可演示 Demo + 数据/指标基线。
阶段二RAG 增强
  • 杭白菊：接入定点相机时序 + 气象做 T3 花蕾计数与产量预测；构建防治 RAG 出农艺建议；MLLM 复核疑难病症。
  • 膳食：上线对话式澄清与份量估计增强；接入膳食指南 RAG 出个性化建议；慢病模块。
  • 产出：可解释报告 + 建议引擎 + 灰度评测体系。
阶段三数据闭环
  • 专家/用户反馈回流 + 主动学习 + 影子评测常态化。
  • 端侧蒸馏部署（现场质检终端 / App 快速预识别）。
  • 知识库热更新机制；多模态补强预留（NIR/深度传感器）。
  • 产出：可持续优化的生产系统 + 数据护城河。

5风险、边界与合规

技术与产品风险

风险对策
病害早期弱信号易漏检定点相机时序对比放大信号 + 多光谱增强 + 召回优先
成分/病原/土壤图像不可判明确边界，输出"疑似+建议复检"，预留多光谱/镜检
产量受气象波动大融合气象数据、给预测区间而非点值
份量估计误差大标准餐具库 + 区间输出 + 对话式补充
MLLM 幻觉RAG 强锚定权威库，数值只取检索结果
MLLM 推理成本/延迟端侧预识别分流，仅疑难样本上云

数据与合规

  • 健康数据敏感：膳食+慢病属个人健康信息，遵循《个人信息保护法》，最小化采集、加密存储、明确授权。
  • 医学免责：营养建议非诊疗意见，慢病场景引导就医，界面明示。
  • 药材鉴定定性：图像结论为辅助判定，法定检验以药检机构为准，报告注明。
  • 知识产权：药典/成分表/指南引用注明来源版本，商用核实授权。
