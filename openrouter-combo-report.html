9 OpenRouter 平台特性 · 怎么用更省

🤖 Auto Router (自动选模型)

设置 model: "openrouter/auto",由 NotDiamond 智能选择。每次响应里会带 model 字段,告诉你实际用了哪个。带 session_id 可锁定会话内一致路由(对 agent 多轮对话很重要)。

// JS SDK
const r = await openRouter.chat.send({
  model: "openrouter/auto",
  session_id: "user-42-conv-7",
  messages: [...]
});
console.log(r.model); // e.g. "anthropic/claude-sonnet-4.5"

🔁 Model Fallbacks (自动失败切换)

在 models 数组里写一组模型,主模型失败(provider down、rate limit、moderation)自动试下一个。OpenRouter 自己实现,不用你写 try/catch。

// OpenAI 兼容 extra_body
{
  "model": "anthropic/claude-opus-4.8",
  "messages": [...],
  "models": [
    "anthropic/claude-opus-4.8",
    "openai/gpt-5.5",
    "moonshotai/kimi-k2.6",
    "deepseek/deepseek-v4-pro"
  ]
}

💸 成本优化 4 件套

  • Prompt Caching:OpenAI cached input 10× 折扣;Anthropic cache reads 0.1×;DeepSeek cache hit 50×
  • Batch API:Anthropic 50% off;Gemini 半价;DeepSeek 也有 batch 折扣
  • Provider 路由:同一模型可能有多个 provider,OpenRouter 自动选最便宜的
  • BYOK 自带 Key:> 1M 月请求可带自己的 provider key,免 OpenRouter 通道费

⚠️ 平台限制 / 坑

  • 信用卡手续费:充值有手续费,大额考虑直接买官方
  • 1M 请求后 BYOK 费:超过 1M 月请求,带自己 key 才省通道费
  • 每跳有 ~10-50ms 延迟:延迟敏感场景考虑 self-hosted (Bifrost 等)
  • 多模态阉割:实测"阉割严重、生图超时 6-10s"——图文业务慎用
