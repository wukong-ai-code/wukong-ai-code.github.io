Hermes 的进化机制 — 每次任务结束，悟空判断本轮经验是「一次性」还是「值得长留」。值得的，会自动升格成 workflows/ 或 skills/ 的活页面，并写进 index.md。所以第二次发同样的需求，质量更高、追问更少。
规则：lowercase-kebab-case · 每个新活页更新 index.md · 一次性内容留在 archive/ · 稳定结论镜像回 Hermes 内存。
对比 · 它和"再多一个 ChatGPT 标签页"有什么不同

真正的差异，藏在三个细节里

✕ 传统聊天助手

  • 每次新对话从零开始介绍自己
  • "我没法访问文件 / 浏览器 / 执行代码"
  • 需要打开网页、登录、切换 IDE
  • 结果是文本，要自己复制粘贴
  • 所有数据上传到别人的服务器
  • 偏好和习惯无法跨会话复用

✓ OpenClaw + 悟空

  • WhatsApp 一句话直接干活，不切环境
  • 真有 browser + filesystem + git，跑出真截图
  • 结果是 .html / .png / .pdf 直接回到聊天
  • 所有数据在你自己的 AWS EC2，MIT 开源
  • 跨会话记得你的偏好（Hermes 三层记忆）
  • 系统级 systemd 服务，24h 在线、断电自启
Rhythm · 一次任务，四个动作

每一次合作，都让下一次更便宜

01 · 接住读真实意图读 memory、读 repo、对齐口径。不假设、不推断模糊词。memory view /memories/repo/
02 · 做成真产物不是"建议你这样做"，是"做完了，文件在这"。create_file v3.html · render_png
03 · 沉淀稳定偏好入库哪类偏好是"以后都这样"，Hermes 自动写到 /memories/。memory create user/style.md
04 · 进化下次更快同样的任务，第二次省掉 60% 的澄清和试错。/standup · 复盘 24h
